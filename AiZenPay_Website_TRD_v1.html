Section 01

Project Overview

AiZenPay is a next-generation AI-powered payment processing company. This website is the company's first public digital presence — Phase 1 of a multi-phase web strategy that will evolve as AiZenPay grows from ISO to Third-Party Processor (TPP) to full Dual-Rail Acquirer-Processor.

Strategic Context — Phase 1

AiZenPay is currently operating as an ISO (Independent Sales Organization) under a confidential acquirer partnership. The website must position AiZenPay as a professional, credible payment processor — without advertising the ISO relationship. The site serves two simultaneous audiences: (1) merchants who need card processing services, and (2) the acquirer partner reviewing AiZenPay's operational readiness for ISO program approval.

Primary Objectives

  • Capture inbound merchant leads via prominent "Get Started" and "Apply Now" CTAs connected to a lead capture form
  • Establish AiZenPay as a legitimate, technology-forward payment processor — not a startup in stealth mode
  • Demonstrate operational readiness to the acquirer partner reviewing the ISO application
  • Build brand foundation (teal/silver/navy identity) that scales through all future phases
  • Plant strategic seeds for future capabilities (stablecoin, AiZenHub platform) without over-promising

What the Website Must NOT Do (Phase 1)

  • Do NOT mention the acquirer partner by name (confidential until ISO agreement is signed)
  • Do NOT claim to be a full Acquirer-Processor or claim direct card brand connections
  • Do NOT make specific stablecoin processing claims as current live capabilities — frame as "coming soon" or "AiZenHub roadmap"
  • Do NOT use language like "ISO" in merchant-facing copy — position as a payment processor, period
  • Do NOT publish specific ARR targets, funding amounts, or investor-level financial data

Brand Identity

Logo

The AiZenPay logo is a teal-and-silver interlocking diamond/shield mark on a dark navy background, with the wordmark "AiZenPay" in which "Ai" appears in teal and "ZenPay" in white. The logo file is provided as AiZenPay_Logo_-_Teal_and_Silver_Design.png.

Logo Usage Rules

Always use on dark backgrounds (navy or near-black). Minimum width: 120px. Maintain aspect ratio. Do not apply color filters. Provide sufficient padding — minimum 16px clearance on all sides. Use the full-color version only; no monochrome variants without approval.

Overall Aesthetic

Dark-mode first. Premium fintech / enterprise SaaS feel. Clean, spacious, confident. Think: Stripe meets a high-end cybersecurity firm. The teal accent pops against the deep navy backgrounds. Silver provides a metallic sophistication. No gradients on white — this is a dark, polished environment.

Section 08

Technical Stack

Phase 1 Stack (Current)

  • HTML5 — Semantic markup, ARIA attributes
  • CSS3 — Custom properties, Grid, Flexbox, no frameworks
  • Vanilla JavaScript — ES6+, no jQuery, no build step required
  • Google Fonts CDN — Syne + DM Sans
  • Netlify Forms — Zero-backend form handling
  • Netlify Hosting — Auto-deploy from GitHub repo

Phase 2 Migration Path (6-12 months)

  • Next.js 14+ — App Router, SSG/ISR
  • React 18 — Component library
  • Tailwind CSS — Utility-first with custom tokens
  • Contentful or Sanity — CMS for blog/content
  • HubSpot Forms SDK — CRM-connected forms
  • Vercel or Netlify — CI/CD deployment

No External Dependencies (Phase 1)

The Phase 1 build must have zero npm dependencies and require no build process. All CSS and JS files are served directly. This ensures Netlify drop-folder deployment works with no configuration, and the site loads fast without a build pipeline.

Browser Support

  • Chrome, Firefox, Safari, Edge — last 2 major versions
  • Mobile: iOS Safari 15+, Android Chrome 100+
  • No IE11 support required
Section 09

Functional Requirements

Navigation

  • Fixed navbar with transparency + blur on scroll (JavaScript scroll listener)
  • Active state on current page nav link
  • Smooth scroll for in-page anchor links
  • Mobile hamburger: slide-in nav drawer, close on outside click or ESC key

Forms (Netlify)

  • All forms use data-netlify="true" for zero-config form handling
  • Add hidden input: <input type="hidden" name="form-name" value="merchant-application">
  • Client-side validation before submit (required fields, email regex, phone regex)
  • Show inline error messages (red border + error text) on invalid submit
  • On success: replace form with thank-you message in same container (no page reload)
  • Honeypot spam field: <input name="bot-field" style="display:none">

Scroll Animations

  • Use IntersectionObserver to add .visible class when elements enter viewport
  • Animate: feature cards (fade up + stagger), stat blocks (count-up number animation), section heroes (fade in)
  • Respect prefers-reduced-motion — skip animations if user has this set

Section 13

Compliance & Legal Requirements

Required Pages

  • privacy.html — Privacy Policy (required before any form captures contact data). Include: data collected, how used, CCPA compliance note, contact for opt-out.
  • terms.html — Terms of Service. Standard website terms. Note: not payment processing merchant agreement — that is a separate document.

Footer Disclaimers

Add to footer bottom bar: "AiZenPay Inc. is a registered payment processing company. PCI DSS Level 1 compliant infrastructure. All trademarks are property of their respective owners."

Form Consent

All lead capture forms must include a checkbox (required) linking to Privacy Policy and Terms of Service before submission is accepted.

Cookie Notice

Minimal cookie banner on first visit: "We use essential cookies to ensure this website works properly. Learn more [Accept]" — Store acceptance in localStorage. No tracking cookies in Phase 1.

Important Note

Phase 1 has no payment processing functionality on the website itself. All processing is handled through AiZenPay's backend systems. The website is a marketing and lead generation platform only. No PCI scope applies to the website.
